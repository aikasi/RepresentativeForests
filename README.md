# 🌲 RepresentativeForests — 명품숲 키오스크

> **대한민국 명품숲 홍보용 인터랙티브 키오스크 어플리케이션**
> 터치 모니터에서 4개의 퀴즈에 답하면, 사용자 성향에 맞는 명품숲 결과 영상과 이미지를 듀얼 모니터로 출력합니다.

## 1. 프로젝트 개요

| 항목                 | 내용                                                                                               |
| -------------------- | -------------------------------------------------------------------------------------------------- |
| **프로젝트명** | RepresentativeForests (명품숲 키오스크)                                                            |
| **용도**       | 전시/홍보용 무인 키오스크                                                                          |
| **디스플레이** | 듀얼 모니터 (메인 모니터: 영상 재생 / 터치 모니터: 퀴즈 UI)                                        |
| **운영 흐름**  | 대기 영상 재생 → 터치로 퀴즈 시작 → 4문항 응답 → 결과 영상 + 결과 이미지 표시 → 자동 대기 복귀 |

### 핵심 기능

- **퀴즈 기반 추천 시스템**: 4문항 퀴즈 결과를 JSON 규칙 파일로 평가하여 15종 이상의 결과 배분
- **크로스페이드 영상 전환**: 두 개의 미디어 플레이어를 핑퐁 방식으로 교차하여 끊김 없는 영상 전환
- **외부 리소스 관리**: 이미지와 영상을 `StreamingAssets` 폴더에서 로드하여, 빌드 없이 콘텐츠 교체 가능
- **프리징 자동 복구**: 영상 정지 감시(Watchdog)로 무인 환경에서도 안정적 운영
- **무입력 자동 복귀**: 설정된 시간 동안 터치가 없으면 자동으로 대기 화면 복귀

## 4. 파일명 규칙

> ⚠️ **중요**: 파일명 규칙을 반드시 준수해야 합니다. 규칙에 맞지 않는 파일은 시스템이 무시하거나 에러를 발생시킵니다.

### 4.1 이미지 파일 (`StreamingAssets/Images/`)

모든 이미지는 **PNG** 형식을 사용합니다.

#### 퀴즈 화면 이미지

| 유형                         | 파일명 패턴            | 예시                                   | 설명                       |
| ---------------------------- | ---------------------- | -------------------------------------- | -------------------------- |
| **퀴즈 배경 이미지**   | `{QQ}-{N}-image.png` | `00-1-image.png`, `01-1-image.png` | 각 퀴즈 화면의 설명 이미지 |
| **선택지 버튼 이미지** | `{QQ}-{N}-btn.png`   | `01-1-btn.png`, `03-4-btn.png`     | 퀴즈 답변 선택 버튼        |

- `{QQ}` = **퀴즈 번호** (2자리, 00부터 시작)
  - `00` = 대기(인트로) 화면
  - `01` = Q1 (1번째 질문)
  - `02` = Q2 (2번째 질문)
  - `03` = Q3 (3번째 질문)
  - `04` = Q4 (4번째 질문)
- `{N}` = **순서 번호** (1부터 시작)
  - 이미지: 해당 퀴즈 화면에 표시되는 이미지 순서
  - 버튼: 선택지 번호 (1=A, 2=B, 3=C, 4=D)

**실제 파일 목록 예시:**

```
00-1-btn.png          ← 대기 화면 시작 버튼
00-1-image.png        ← 대기 화면 설명 이미지
00-2-image.png        ← 대기 화면 추가 이미지
01-1-btn.png          ← Q1 선택지 A 버튼
01-1-image.png        ← Q1 퀴즈 설명 이미지
01-2-btn.png          ← Q1 선택지 B 버튼
02-1-btn.png          ← Q2 선택지 A 버튼
02-2-btn.png          ← Q2 선택지 B 버튼
03-1-btn.png          ← Q3 선택지 A 버튼
03-2-btn.png          ← Q3 선택지 B 버튼
03-3-btn.png          ← Q3 선택지 C 버튼
03-4-btn.png          ← Q3 선택지 D 버튼
04-1-btn.png          ← Q4 선택지 A 버튼
04-2-btn.png          ← Q4 선택지 B 버튼
04-3-btn.png          ← Q4 선택지 C 버튼
04-4-btn.png          ← Q4 선택지 D 버튼
```

#### 결과 이미지

| 유형                       | 파일명 패턴         | 예시                                 | 설명                                  |
| -------------------------- | ------------------- | ------------------------------------ | ------------------------------------- |
| **결과 배경 이미지** | `{NN}-Result.png` | `01-Result.png`, `15-Result.png` | 퀴즈 결과 화면에 표시되는 배경 이미지 |

- `{NN}` = **결과 카테고리 ID** (2자리, 01부터 시작)
- `QuizRules.json`의 `imageId` 값과 매칭됩니다.
- 시스템은 내부적으로 `{imageId}-Result.png` 형식으로 파일을 탐색합니다.

**현재 등록된 결과 이미지:**


### 4.2 동영상 파일 (`StreamingAssets/Videos/`)

지원 확장자: `.mp4`, `.mov`, `.avi`, `.wmv`
**(권장: `.mp4` — H.264 코덱)**

#### 대기 영상

| 유형                        | 파일명 패턴 | 예시 | 설명                                            |
| --------------------------- | ----------- | ---- | ----------------------------------------------- |
| **대기(아이들) 영상** | `00.mp4`  | —   | 터치 대기 중 메인 모니터에서 반복 재생되는 영상 |

- 파일명이 `0`으로만 구성된 경우 대기 영상으로 인식합니다.
- **반드시 1개만 존재해야 합니다.** (중복 시 마지막 파일 사용, 경고 발생)

#### 결과 영상

| 유형                | 파일명 패턴                     | 예시                       | 설명                                    |
| ------------------- | ------------------------------- | -------------------------- | --------------------------------------- |
| **결과 영상** | `{카테고리ID}-{재생순서}.mp4` | `01-1.mp4`, `02-5.mp4` | 퀴즈 결과에 따라 순서대로 재생되는 영상 |

- `{카테고리ID}` = 결과 카테고리 번호 (숫자)
  - `QuizRules.json`의 `videoId` 값과 매칭됩니다.
- `{재생순서}` = 해당 카테고리 내 영상 재생 순서 (1부터 시작)
  - 같은 카테고리 ID를 가진 영상이 여러 개이면 순서 번호 기준으로 자동 정렬되어 연속 재생됩니다.

**실제 파일 목록 예시:**

```
00.mp4                ← 대기 영상 (루프 재생)

01-1.mp4              ← 카테고리 01 (100대 명품숲), 1번째 영상
01-2.mp4              ← 카테고리 01, 2번째 영상 (01-1 끝나면 자동 전환)

02-1.mp4              ← 카테고리 02 (명품숲길), 1번째 영상
02-2.mp4              ← 카테고리 02, 2번째 영상
02-3.mp4              ← 카테고리 02, 3번째 영상
02-4.mp4              ← 카테고리 02, 4번째 영상
02-5.mp4              ← 카테고리 02, 5번째 영상 (마지막 → 대기 영상 복귀)

03-1.mp4              ← 카테고리 03 (자연휴양림)

04-1.mp4              ← 카테고리 04 (정원), 1번째 영상
04-2.mp4              ← 카테고리 04, 2번째 영상
```

#### 영상 파일 무결성 검사

시스템은 기동 시 `MediaScanner`가 자동으로 아래 항목을 검사합니다:

| 검사 항목                           | 결과                           |
| ----------------------------------- | ------------------------------ |
| 0KB (빈) 파일 존재                  | ❌ 치명적 에러 → 앱 진입 차단 |
| 같은 이름에 확장자만 다른 중복 파일 | ❌ 치명적 에러 → 앱 진입 차단 |
| 대기 영상(`00.mp4`) 미존재        | ❌ 치명적 에러 → 앱 진입 차단 |
| 파일명 규칙 불일치                  | ⚠️ 경고 로그 출력 후 무시    |

## 5. 설정 파일 가이드

### 5.1 Settings.txt (키오스크 환경 설정)

**위치**: 에디터 모드에서는 `Assets/Settings.txt`, 빌드에서는 `.exe` 파일과 같은 폴더에 위치합니다.
**형식**: CSV (쉼표 구분, `키, 값, 설명(선택)`)
**주의**: 쉼표가 없는 줄은 자동으로 무시됩니다 (주석으로 활용 가능).

| 키                    | 기본값    | 설명                                       |
| --------------------- | --------- | ------------------------------------------ |
| `ResolutionX`       | `1920`  | 메인 모니터 가로 해상도                    |
| `ResolutionY`       | `1080`  | 메인 모니터 세로 해상도                    |
| `UseSubMonitor`     | `false` | 서브(터치) 모니터 사용 여부                |
| `SubResolutionX`    | `1920`  | 서브 모니터 가로 해상도                    |
| `SubResolutionY`    | `1080`  | 서브 모니터 세로 해상도                    |
| `FullScreen`        | `true`  | 전체 화면 모드                             |
| `HideCursor`        | `false` | 마우스 커서 숨기기                         |
| `CrossfadeTime`     | `1.0`   | 영상 크로스페이드 전환 시간 (초)           |
| `PreloadDelayTime`  | `2.0`   | 백그라운드 영상 사전 로딩 지연 시간 (초)   |
| `WatchdogTimeout`   | `5.0`   | 영상 프리징 감지 기준 시간 (초)            |
| `MaxRetryCount`     | `3`     | 프리징 복구 최대 재시도 횟수               |
| `InactivityTimeout` | `60.0`  | 무입력 자동 복귀 시간 (초, 0이면 비활성화) |

**설정 파일 예시:**

```
이 아래부터는 화면 관련 설정입니다 (컴마가 없으면 시스템이 줄 전체를 무시합니다)
ResolutionX, 1920, 가로 해상도를 말합니다
ResolutionY, 1080, 세로 해상도를 말합니다
UseSubMonitor, true
SubResolutionX, 1920
SubResolutionY, 1080
FullScreen, true, 전체화면 여부 (true 또는 false로 입력하세요)
HideCursor,false, 마우스감추는 효과 적용 여부
--------- 비디오 설정 ---------
CrossfadeTime, 1.0, 부드럽게 전환되는 시간(초)
WatchdogTimeout, 5.0, 영상이 멈췄는지 감시하는 시간 기준
```

### 5.2 QuizRules.json (퀴즈 결과 규칙)

**위치**: `StreamingAssets/QuizRules.json`

이 파일로 **코드 수정 없이** 퀴즈 결과 분기를 변경할 수 있습니다.

**구조:**

```json
{
  "totalQuestions": 4,
  "fallbackVideoId": "01",
  "fallbackImageId": "01",
  "rules": [
    {
      "priority": 1,
      "ruleName": "규칙 이름 (관리용)",
      "conditions": ["A", "A", "A", "ANY"],
      "videoId": "04",
      "imageId": "12"
    }
  ]
}
```

| 필드                | 설명                                                                     |
| ------------------- | ------------------------------------------------------------------------ |
| `totalQuestions`  | 총 퀴즈 문항 수                                                          |
| `fallbackVideoId` | 모든 규칙에 매칭되지 않을 때 사용할 영상 ID                              |
| `fallbackImageId` | 모든 규칙에 매칭되지 않을 때 사용할 이미지 ID                            |
| `priority`        | 규칙 우선순위 (숫자가 작을수록 먼저 검사)                                |
| `ruleName`        | 규칙 설명 (로그 출력/관리용, 시스템 동작에 영향 없음)                    |
| `conditions`      | 각 문항별 매칭 조건 배열 (`"A"`, `"B"`, `"C"`, `"D"`, `"ANY"`) |
| `videoId`         | 매칭 시 재생할 영상 카테고리 ID (`Videos/{videoId}-*.mp4`)             |
| `imageId`         | 매칭 시 표시할 결과 이미지 ID (`Images/{imageId}-Result.png`)          |

> **`"ANY"` 와일드카드**: 해당 문항의 모든 응답을 허용합니다.

### 시스템 흐름도

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  CSVReader   │────▶│  GameManager │────▶│ VideoManager │
│ (설정 로드)   │     │  (시스템 브릿지)│     │(영상 재생/전환)│
└──────────────┘     └──────┬───────┘     └──────▲───────┘
                            │                     │
                     ┌──────▼───────┐     ┌───────┴──────┐
                     │ QuizManager  │────▶│ MediaScanner │
                     │ (퀴즈 평가)   │     │ (영상 스캔)   │
                     └──────┬───────┘     └──────────────┘
                            │
                     ┌──────▼───────┐     ┌──────────────┐
                     │  UIManager   │────▶│ ImageManager │
                     │ (패널 전환)   │     │(이미지 캐시)  │
                     └──────────────┘     └──────────────┘
```

### 스크립트 목록 및 역할

| 스크립트                | 경로                              | 역할                                                                 |
| ----------------------- | --------------------------------- | -------------------------------------------------------------------- |
| **CSVReader**     | `Scripts/CSVReader.cs`          | `Settings.txt`를 파싱하여 전역 Dictionary에 저장 (실행 순서 -1000) |
| **GameManager**   | `Scripts/Core/GameManager.cs`   | 해상도/커서 등 초기 설정 적용, 퀴즈→영상 연결 브릿지                |
| **MediaScanner**  | `Scripts/Core/MediaScanner.cs`  | `Videos/` 폴더 스캔, 무결성 검사, 영상 경로 캐싱                   |
| **QuizManager**   | `Scripts/Quiz/QuizManager.cs`   | 사용자 답변 수집, JSON 규칙 매칭, 결과 ID 도출                       |
| **UIManager**     | `Scripts/UI/UIManager.cs`       | 터치 모니터 패널 전환, 터치 잠금, 무입력 자동 복귀                   |
| **ImageManager**  | `Scripts/UI/ImageManager.cs`    | 참조 카운팅 기반 이미지 로드/해제 (VRAM 관리)                        |
| **ExternalImage** | `Scripts/UI/ExternalImage.cs`   | UI 오브젝트에 부착하여 외부 이미지 자동 로드/해제                    |
| **ErrorPopup**    | `Scripts/UI/ErrorPopup.cs`      | 에러 팝업 큐 관리, 페이드 애니메이션                                 |
| **VideoManager**  | `Scripts/Video/VideoManager.cs` | AVPro 핑퐁 크로스페이드, 시퀀스 재생, 사전 로딩                      |
| **VideoWatchdog** | `Scripts/VideoWatchdog.cs`      | 영상 프리징 감지 및 자동 복구                                        |
| **Logger**        | `Scripts/Logger.cs`             | 비동기 파일 로그 기록 (`PersistentDataPath/Logs/`)                 |

### 실행 순서 (Execution Order)

```
1. CSVReader         (Awake, -1000)  ← Settings.txt 파싱
2. 모든 싱글톤       (Awake, 기본)    ← Instance 초기화
3. MediaScanner      (Start)          ← Videos 폴더 스캔
4. GameManager       (Start)          ← 해상도 적용 + 이벤트 구독
5. VideoManager      (Start)          ← 대기 영상 재생 시작
6. UIManager         (Start)          ← UI 패널 초기화
7. QuizManager       (Awake)          ← QuizRules.json 로드
```

## 7. 빌드 및 배포

### 배포 시 파일 구성

빌드 후 납품할 때 아래와 같은 구조로 전달합니다:

```
배포 폴더/
├── RepresentativeForests.exe           # 실행 파일
├── RepresentativeForests_Data/         # Unity 데이터 폴더
│   ├── StreamingAssets/                # ★ 외부 리소스
│   │   ├── Images/                     #   이미지 파일들
│   │   ├── Videos/                     #   영상 파일들
│   │   └── QuizRules.json             #   퀴즈 규칙
│   └── ...
├── Settings.txt                        # ★ 환경 설정 파일
├── MonoBleedingEdge/                   # Mono 런타임
└── UnityPlayer.dll                     # Unity 엔진
```

### 배포 주의사항

- **Settings.txt**는 `.exe` 파일과 **같은 폴더**에 위치해야 합니다.
- **StreamingAssets** 폴더의 이미지/영상은 빌드 후에도 자유롭게 교체할 수 있습니다.
- 영상 교체 시 반드시 [파일명 규칙](#4-파일명-규칙-)을 준수해야 합니다.
- `QuizRules.json`도 빌드 없이 편집 가능합니다 (텍스트 편집기로 수정).

## 8. 유지보수 가이드

### 콘텐츠 교체 (빌드 불필요)

| 작업                             | 방법                                                                  |
| -------------------------------- | --------------------------------------------------------------------- |
| **결과 영상 교체**         | `StreamingAssets/Videos/` 폴더의 해당 파일을 같은 이름으로 덮어쓰기 |
| **결과 영상 추가**         | 같은 카테고리 ID에 순서 번호를 추가 (예:`02-6.mp4`)                 |
| **결과 이미지 교체**       | `StreamingAssets/Images/` 폴더의 해당 파일을 같은 이름으로 덮어쓰기 |
| **퀴즈 결과 분기 변경**    | `QuizRules.json` 편집 (규칙 추가/수정/삭제)                         |
| **해상도 변경**            | `Settings.txt`의 `ResolutionX`, `ResolutionY` 수정              |
| **크로스페이드 속도 변경** | `Settings.txt`의 `CrossfadeTime` 수정                             |

### 새로운 결과 카테고리 추가 절차

1. 영상 파일을 `Videos/` 폴더에 추가 (예: `05-1.mp4`, `05-2.mp4`)
2. 결과 이미지를 `Images/` 폴더에 추가 (예: `16-Result.png`)
3. `QuizRules.json`에 새 규칙 추가:
   ```json
   {"priority": 21, "ruleName": "새 카테고리", "conditions": ["B","A","D","D"], "videoId": "05", "imageId": "16"}
   ```
4. 앱 재시작 (빌드 불필요)

### 로그 파일 위치

- **Windows**: `C:\Users\{사용자명}\AppData\LocalLow\Metadevs\RepresentativeForests\Logs\`
- 파일명 형식: `YYMMDD-HHmmss.log`
- 오래된 로그는 최대 1000개까지 유지 후 자동 삭제됩니다.

## 9. 트러블슈팅

| 증상                        | 원인                           | 해결 방법                                        |
| --------------------------- | ------------------------------ | ------------------------------------------------ |
| 앱 시작 시 에러 팝업 발생   | 영상 파일 0KB 또는 중복 감지   | `Videos/` 폴더의 손상/중복 파일 제거           |
| 결과 이미지가 표시되지 않음 | 파일명 규칙 불일치             | `{imageId}-Result.png` 형식 확인               |
| 영상이 재생되지 않음        | 코덱 미지원 또는 파일 손상     | H.264 코덱의 `.mp4`로 재인코딩                 |
| 영상이 멈춤 (프리징)        | 하드웨어/디스크 성능 부족      | `WatchdogTimeout` 조정, 영상 비트레이트 낮추기 |
| 터치가 안 됨                | 크로스페이드 중 터치 잠금 상태 | 전환 완료 후 자동 해제됨. 지속 시 앱 재시작      |
| 서브 모니터 미표시          | 물리적 모니터 미연결           | Windows 디스플레이 설정에서 확장 모드 확인       |
| 대기 영상 재생 안 됨        | `00.mp4` 파일 누락           | `Videos/` 폴더에 `00.mp4` 배치               |
